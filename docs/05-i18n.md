# 05 — Internationalization (i18n)

## Languages

- **Default:** Bulgarian (`bg`)
- **Optional:** English (`en`)

Bulgarian is the primary language. English exists for two reasons:

1. The customer-facing kiosk in tourist-heavy areas (Sofia center, Sunny Beach, Plovdiv)
2. The rare English-speaking owner who wants to manage their restaurant in English

## Library

`next-intl` — chosen because:

- First-class App Router support
- Server component compatible
- Type-safe message keys
- Pluralization and date/number formatting built in

## File structure

```
lib/i18n/
├── config.ts              # Locale config, default locale
├── request.ts             # next-intl request config
└── messages/
    ├── bg.json            # Bulgarian (source of truth)
    └── en.json            # English translations
```

## Translation rules

### 1. Bulgarian first, always

Write the Bulgarian copy first. It's the source of truth. The English version is a translation, not the other way around.

### 2. Don't translate idioms literally

Bad: `"You're killing it!"` → `"Убиваш го!"` (sounds violent, not enthusiastic in BG)
Good: `"You're killing it!"` → `"Браво!"` or `"Супер си!"`

### 3. Length matters

Bulgarian is ~20% longer than English on average. Design components to handle text overflow gracefully:

- Buttons: avoid fixed widths, use min-width + padding
- Headings: test with longest BG version
- Mobile: be especially careful with cramped layouts

### 4. Cyrillic-aware fonts

The app uses Instrument Serif for display, Inter for UI, and JetBrains Mono for mono labels. Bulgarian UI copy should render through Inter or another tested Cyrillic-capable fallback. Instrument Serif is brand/display flavor, not the default reading face for long Cyrillic text.

When touching `app/layout.tsx` or `next/font` setup, preserve Cyrillic coverage for UI text and test real Bulgarian strings on Windows and Android. Never rely on a font without Cyrillic glyphs for customer-facing copy.

### 5. Date and number formatting

Use `next-intl`'s formatters, not raw JS:

```ts
import { useFormatter } from "next-intl";

const format = useFormatter();
format.dateTime(date, { year: "numeric", month: "long", day: "numeric" });
// BG: "17 април 2026 г."
// EN: "April 17, 2026"

format.number(price, { style: "currency", currency: "BGN" });
// BG: "19,99 лв."
// EN: "BGN 19.99"
```

Bulgarian decimal separator is comma, not period. Bulgarian currency display puts "лв." after the number.

### 6. Pluralization

Bulgarian has three plural forms (one, few, other). Use ICU MessageFormat:

```json
{
  "feedback.count": "{count, plural, =0 {Все още няма отзиви} one {1 отзив} other {# отзива}}"
}
```

## Customer language detection in kiosk

The kiosk shows a small language toggle in the corner (БГ / EN). It defaults to the restaurant's `language_default`.

The customer can switch with one tap. Selection is per-session, not persisted.

If `restaurant.customer_languages` only contains `['bg']`, the toggle is hidden entirely.

## Owner language

Set during signup, stored in `auth.users.user_metadata.language`. Owner can change it in `/dashboard/settings`.

The dashboard, all admin UI, all emails, and all push notifications respect this setting.

## What does NOT get translated

- Restaurant names (user-generated)
- Menu item names (separate `name_bg` and `name_en` columns)
- Customer comments (stored as-is in their language)
- Receipt raw text (extracted as-is)

## Translation workflow

1. Developer adds new BG keys to `bg.json` while building
2. Periodically (weekly), copy missing keys to `en.json` and add English translations
3. Pre-launch: have a fluent English speaker review the entire `en.json`
4. Post-launch: missing translations fall back to BG (graceful degradation)

## i18n config

```ts
// lib/i18n/config.ts
export const locales = ["bg", "en"] as const;
export const defaultLocale = "bg";
export type Locale = (typeof locales)[number];

export const localeNames: Record<Locale, string> = {
  bg: "Български",
  en: "English",
};
```

```ts
// lib/i18n/request.ts
import { getRequestConfig } from "next-intl/server";
import { locales, defaultLocale } from "./config";

export default getRequestConfig(async ({ locale }) => {
  if (!locales.includes(locale as any)) {
    locale = defaultLocale;
  }

  return {
    messages: (await import(`./messages/${locale}.json`)).default,
  };
});
```

## Routing

We do NOT use locale-prefixed routes (`/bg/dashboard`, `/en/dashboard`) for the dashboard. Owners pick their language once in settings and that's it.

The kiosk uses a query param (`/kiosk?lang=en`) for the in-session language switch.

The marketing site uses locale-prefixed routes (`/`, `/en/`) for SEO benefits in the English market later.

## Currency

All prices stored as `numeric` in BGN. Display:

- BG locale: `19,99 лв.`
- EN locale: `BGN 19.99` or `~€10.20` (with conversion display)

Subscription pricing is in EUR (€10/month) because that's how Stripe is set up. Display:

- BG locale: `€10/месец (~20 лв.)`
- EN locale: `€10/month`
